# Decision records — index

One row per ADR. **Check the Status column before applying any decision** — a superseded ADR reads
as authoritative if you open it without this context. Partial supersessions name the items
affected; everything not named remains in force.

**Maintenance:** when you create an ADR, add its row here. When an ADR supersedes another (fully or
partially), update **both** rows — the new ADR's row and the superseded one's Status. Number ADRs
sequentially (`0001`, `0002`, …); never renumber or reuse a number.
